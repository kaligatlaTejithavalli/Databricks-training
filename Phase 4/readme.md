# Phase 4 – Business Pipeline & Analytics using PySpark

---

## Overview

This project demonstrates the implementation of an end-to-end business analytics pipeline using **PySpark**. The pipeline performs data ingestion, cleaning, transformation, aggregation, customer segmentation, and reporting by integrating customer and sales datasets. Each analytical task is implemented using both **Spark SQL** and the **PySpark DataFrame API** to illustrate how SQL queries can be translated into scalable distributed data processing.

---

## Objectives

- Build an end-to-end ETL and analytics pipeline.
- Perform data cleaning before analysis.
- Join multiple datasets using common keys.
- Generate business insights from transactional data.
- Implement SQL and equivalent PySpark solutions.
- Create a reusable business reporting pipeline.

## Project Workflow

### 1. Data Ingestion

- Read customer dataset.
- Read sales dataset.

### 2. Data Cleaning

- Remove rows with null `customer_id`.
- Remove duplicate records.
- Filter invalid transaction amounts.
- Validate data schema.

### 3. Data Integration

- Join customer and sales datasets using `customer_id`.

### 4. Business Analytics

#### Task 1 – Daily Sales

Calculate the total sales generated for each day.

**Output**

| Sale Date | Total Sales |
|-----------|-------------|

---

#### Task 2 – City-wise Revenue

Calculate the total revenue generated by each city.

**Output**

| City | Total Revenue |
|------|---------------|

---

#### Task 3 – Top 5 Customers

Identify the top five customers based on total spending.

**Output**

| Customer Name | Total Spend |
|---------------|-------------|

---

#### Task 4 – Repeat Customers

Identify customers who placed more than one order.

**Output**

| Customer ID | Order Count |
|-------------|-------------|

---

#### Task 5 – Customer Segmentation

Categorize customers based on their total spending.

**Business Rules**

- 🥇 **Gold** → Total Spend > 10000
- 🥈 **Silver** → Total Spend between 5000 and 10000
- 🥉 **Bronze** → Total Spend < 5000

**Output**

| Customer Name | Total Spend | Segment |
|---------------|-------------|---------|

---

#### Task 6 – Final Reporting Table

Generate a consolidated report containing:

- Customer Name
- City
- Total Spend
- Order Count
- Customer Segment

---

#### Task 7 – Save Output

Save the final reporting table for future analysis.

Each business task is implemented using:

- Equivalent PySpark DataFrame API

This demonstrates how common SQL operations—including filtering, grouping, joining, aggregations, and conditional logic—can be implemented efficiently using PySpark transformations.
